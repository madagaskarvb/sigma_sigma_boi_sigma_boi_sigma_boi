# Contributing

## Как сдаем задания
1. Создай ветку от main:
   git checkout -b feat/<short-task-name>
2. Внеси изменения, проверь запуск/сборку.
3. Сделай коммиты (осмысленно):
   feat: ...
   fix: ...
   docs: ...
   chore: ...
4. Запушь ветку и открой Pull Request в main.

## Стандарты сервисов
Каждый сервис обязан иметь:
- /health
- /version
- Dockerfile
- .env.example
- README.md (как запустить)

## Запрещено
- пушить секреты (.env)
- ломать общую структуру каталогов без согласования
